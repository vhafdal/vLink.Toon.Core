# Contributing to vLink

Thanks for your interest in contributing.

We welcome improvements, bug fixes, features, and documentation updates.

---

## 🚀 How to Contribute

1. Fork the repository
2. Create a branch:  
  git checkout -b feature/your-feature-name
3. Make your changes
4. Commit:  
  git commit -m "Add: short description"
5. Push:  
  git push origin feature/your-feature-name
6. Open a Pull Request

---

## 📋 Guidelines

- Keep changes focused and minimal
- Follow existing code style
- Write clear commit messages
- Add tests if relevant
- Update documentation if needed

---

## ⚖️ Contributor License Agreement (CLA)

Before your contribution can be accepted, you must agree to the CLA.

See: [CLA.md](https://docs.vlink.is/share/j023b3g57l/p/cla-thb1d9o9nY)

By submitting a contribution, you agree that:

- Your contribution is assigned to vLink
- vLink has full rights to use, modify, and relicense it
- vLink may use it in commercial products

If you do not agree, do not submit a contribution.

---

## 🔍 Pull Request Process

- Your PR will be reviewed
- Changes may be requested
- Once approved, it will be merged

## 🧪 Testing
